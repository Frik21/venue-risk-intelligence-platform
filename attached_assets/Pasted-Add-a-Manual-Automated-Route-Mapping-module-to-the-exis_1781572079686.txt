Add a Manual & Automated Route Mapping module to the existing Venue Risk Intelligence Platform.

The platform already includes venue assessments, OSINT, mapping, evidence, monitoring, and incident intelligence. Build this feature on top of the existing system.

FEATURE: Manual & Automated Route Mapping

Purpose:
Allow users to create, save, edit, verify, and monitor operational routes linked to venues and assessments.

Route Types:
- Primary Extraction Route
- Secondary Extraction Route
- Medical Evacuation Route
- VIP Arrival Route
- VIP Departure Route
- Staff Access Route
- Supplier / Service Route
- Emergency Services Access Route

Route Creation Methods:

1. Street Name Route Builder
Users can enter a start point, end point, and preferred street names or road segments.

System must:
- Accept named streets in sequence
- Attempt to geocode/resolve each street
- Generate a route using those streets where possible
- Flag unresolved streets
- Allow user correction
- Save route to assessment

2. Endpoint Marker Routing
Users can place or search:
- Start marker
- End marker
- Optional waypoint markers

System must:
- Generate automated route
- Show estimated distance
- Show estimated travel time
- Allow route adjustment
- Save route as selected route type

3. Freehand Route Drawing
Users can manually draw a route on the map.

System must:
- Store drawn route geometry as GeoJSON
- Allow route editing
- Allow notes
- Calculate approximate route distance
- Mark route creation method as manual_draw

Route Data Fields:
- Route name
- Route type
- Assessment ID
- Venue ID
- Creation method
- Start label
- Start latitude
- Start longitude
- End label
- End latitude
- End longitude
- Waypoints JSON
- Route geometry GeoJSON
- Estimated distance
- Estimated travel time
- Constraints
- Analyst notes
- Verified status
- Created by
- Created at
- Updated at

Route Constraints:
- Traffic congestion
- Road closures
- Protest activity
- Construction
- Narrow roads
- One-way streets
- Limited turning space
- Security exposure points
- Poor lighting
- High pedestrian density
- Ride-share clustering
- Limited law enforcement presence

OSINT Route Corridor Analysis:
After a route is created, analyze the route corridor for:
- Crime incidents near route
- Protest activity near route
- Civil unrest near route
- Road closures
- Construction
- Police advisories
- Government alerts
- Traffic disruption

The system should flag findings for analyst review.

Important rule:
Automated route and OSINT findings may suggest route risks, but route approval must be manual and analyst verified.

Add a Route Planning panel inside the map view and assessment view.

Panel actions:
